# 项目规则

## 语言规范

- **所有文档必须使用中文**：包括设计规范、README、注释等
- **代码注释和命名**：使用英文（遵循国际惯例）

## 目录结构

```
项目根目录/
├── docs/                  # 文档目录
│   └── superpowers/
│       └── specs/         # 设计规范
├── src/                   # 源代码目录
│   ├── css/               # 样式文件
│   └── js/                # JavaScript 文件
└── index.html             # 入口文件
```

## 规范要求

1. 所有非代码文件必须使用中文编写
2. 代码中的注释可以使用中文或英文
3. 保持代码简洁，无需不必要的注释

## TDD 开发模式

- **先写测试，再写实现代码**
- 测试必须先失败，然后编写最小化代码使其通过
- 遵循 Red-Green-Refactor 循环：
  1. RED：编写失败的测试
  2. GREEN：编写最小化代码使测试通过
  3. REFACTOR：重构优化代码
- 测试文件放在 `src/js/__tests__/` 目录下
- 使用 Jest 作为测试框架
