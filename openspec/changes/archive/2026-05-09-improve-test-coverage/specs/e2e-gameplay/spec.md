## 新增需求

### 需求：E2E 测试应验证游戏初始化

页面加载时，游戏应渲染出完整可玩的界面。

#### 场景：页面加载后显示游戏网格
- **当** 加载 index.html 页面时
- **那么** 游戏容器应可见
- **那么** 网格应包含 81 个单元格（初级难度 9x9）
- **那么** 难度选择器按钮应可见
- **那么** 地雷计数器应显示 '010'
- **那么** 计时器应显示 '000'
- **那么** 笑脸按钮应可见

### 需求：E2E 测试应验证单元格交互

用户应能点击单元格揭开它，右键点击来标记旗帜。

#### 场景：左键点击揭开安全单元格
- **当** 用户左键点击一个不包含地雷的单元格时
- **那么** 该单元格的 class 应包含 'revealed'
- **那么** 如果该单元格周围有地雷，应显示数量

#### 场景：右键点击标记旗帜
- **当** 用户右键点击一个未打开的单元格时
- **那么** 该单元格的 class 应包含 'flagged'
- **那么** 单元格中应显示旗帜图标

#### 场景：右键点击取消已标记的旗帜
- **当** 用户右键点击一个已标记旗帜的单元格时
- **那么** 该单元格的 class 不再包含 'flagged'
- **那么** 旗帜图标应被移除

#### 场景：点击已揭开的单元格无变化
- **当** 用户点击一个已揭开的单元格时
- **那么** 游戏状态不应改变
- **那么** 不应抛出任何错误

### 需求：E2E 测试应验证难度切换

用户应能在初级、中级、高级难度之间切换。

#### 场景：切换到中级难度
- **当** 用户点击 '中级' 难度按钮时
- **那么** 网格应包含 256 个单元格（16x16）
- **那么** 地雷计数器应更新为 '040'

#### 场景：切换到高级难度
- **当** 用户点击 '高级' 难度按钮时
- **那么** 网格应包含 480 个单元格（16x30）
- **那么** 地雷计数器应更新为 '099'

#### 场景：切换回初级难度
- **当** 用户点击 '初级' 难度按钮时
- **那么** 网格应包含 81 个单元格（9x9）
- **那么** 地雷计数器应更新为 '010'

### 需求：E2E 测试应验证笑脸按钮重置

笑脸按钮应将游戏重置为初始状态。

#### 场景：点击笑脸按钮重置游戏
- **当** 用户点击笑脸按钮时
- **那么** 所有单元格应回到未打开状态
- **那么** 地雷计数器应重置为难度默认值
- **那么** 计时器应重置为 '000'

### 需求：E2E 测试应验证计时器行为

计时器应在首次点击时启动，在游戏结束或胜利时停止。

#### 场景：首次点击时计时器启动
- **当** 用户第一次点击单元格时
- **那么** 1 秒后，计时器应显示大于 '000' 的值

#### 场景：游戏结束时计时器停止
- **当** 用户踩中地雷时
- **那么** 计时器应停止递增
- **那么** 笑脸图标应变为 '失败' 状态

### 需求：E2E 测试应验证首次点击安全保护

首次点击的单元格绝不会包含地雷。

#### 场景：首次点击永远安全
- **当** 用户将任何单元格作为首次操作点击时
- **那么** 游戏不应结束
- **那么** 被点击位置不应显示地雷

### 需求：E2E 测试应验证主题在页面刷新后持久化

选中的主题在页面重新加载后应保持。

#### 场景：刷新页面后主题保持
- **当** 用户选择一个非默认主题（如 'classic'）时
- **当** 页面被重新加载时
- **那么** 选中的主题应仍然生效
- **那么** 主题名称显示应显示之前选中的主题
