## ADDED Requirements

### Requirement: 主题选择器 UI

游戏面板必须在头部区域显示主题选择器下拉菜单。

主题选择器必须：
- 位于游戏头部右上角区域
- 显示当前主题名称
- 点击时显示包含所有 5 个主题的下拉列表
- 在下拉列表中高亮当前激活的主题

#### Scenario: 页面加载时主题选择器渲染
- **WHEN** 游戏页面加载时
- **THEN** 游戏头部应可见主题选择器元素
- **AND** 应显示当前激活主题的名称

#### Scenario: 主题选择器下拉菜单打开
- **WHEN** 用户点击主题选择器按钮时
- **THEN** 应出现下拉菜单
- **AND** 应列出所有 5 个可用主题
- **AND** 当前激活主题应被视觉上高亮

#### Scenario: 选择后下拉菜单关闭
- **WHEN** 用户点击下拉菜单中的主题时
- **THEN** 下拉菜单应关闭
- **AND** 选择器按钮应显示新选择的主题名称

#### Scenario: 点击外部区域关闭下拉菜单
- **WHEN** 下拉菜单处于打开状态
- **AND** 用户点击下拉菜单区域外部时
- **THEN** 下拉菜单应关闭

### Requirement: 主题视觉应用

当用户选择主题时，系统必须应用该主题的 CSS 变量和图标。

当主题被应用时：
- `<html>` 元素的 class 应设置为主题对应的 className
- 所有显示地雷图标的单元格应更新为该主题的地雷图标
- 所有已标旗单元格应更新为该主题的旗帜图标
- 表情按钮应更新为对应状态的图标
- 视觉样式应平滑过渡（CSS transition 应用于颜色/背景属性）

#### Scenario: 主题改变单元格外观
- **WHEN** 应用 'candy' 主题时
- **THEN** 未揭示的单元格应使用糖果主题颜色（粉色/圆角）
- **AND** 视觉外观应与深色主题不同

#### Scenario: 主题立即改变图标
- **WHEN** 主题从 'dark' 切换到 'space' 时
- **THEN** 所有可见地雷图标应从 '💣' 变为 '🌑'
- **AND** 所有可见旗帜图标应从 '🚩' 变为 '⭐'

#### Scenario: 主题切换平滑
- **WHEN** 应用主题时
- **THEN** 颜色变化应以 0.3s 动画过渡
- **AND** 不应有布局偏移或未样式化内容的闪烁

#### Scenario: 表情随游戏状态和主题更新
- **WHEN** 游戏处于进行状态且主题为 'ocean' 时
- **THEN** 表情应显示 '🐠'
- **WHEN** 用户获胜时
- **THEN** 表情应变为 '🐬'
