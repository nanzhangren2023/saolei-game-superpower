# 变更：为扫雷游戏添加主题支持

## 问题

当前扫雷游戏只有一个固定的深色主题样式，所有颜色和图标都硬编码在 CSS 和 HTML 中。这限制了用户的个性化选择，也不符合商用项目的可扩展性要求。

## 解决方案

实现一个完整的主题系统，支持 5 个预设主题的切换，包括：

- **深色经典**（默认）- 当前深蓝/紫渐变风格
- **Windows 经典** - 怀旧 Windows 95 风格（3D 边框，灰色背景）
- **太空探险** - 科幻深空风格（紫罗兰强调色）
- **海洋世界** - 清新蓝色海洋风格
- **糖果派对** - 可爱粉色/彩虹风格

### 核心特性

1. **主题切换 UI** - 游戏面板右上角下拉菜单
2. **CSS 变量系统** - 所有颜色使用 CSS Custom Properties
3. **图标系统** - 每个主题包含独立的图标映射（emoji）
4. **持久化** - 使用 localStorage 保存用户选择
5. **平滑过渡** - 主题切换时的 CSS 动画
6. **E2E 测试** - Playwright 测试覆盖

### 技术架构

```
目录结构:
├── src/
│   ├── css/
│   │   ├── base.css          ← 基础重置
│   │   ├── themes.css        ← 所有主题 CSS 变量
│   │   ├── components.css    ← 使用变量的组件样式
│   │   └── animations.css    ← 动画定义
│   └── js/
│       ├── game.js           ← 扫雷核心逻辑
│       ├── renderer.js       ← 渲染逻辑
│       ├── theme-manager.js  ← 主题管理器
│       └── __tests__/        ← 单元测试
├── e2e/
│   ├── pages/
│   │   └── game-page.js      ← 页面对象
│   └── tests/
│       ├── theme-switch.spec.js
│       └── persistence.spec.js
└── index.html                ← 精简 HTML
```

## 影响范围

### 文件变更
- **重构** - 分离内联 JS/CSS 到独立文件
